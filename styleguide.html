Styleguide

Prefixes legend:

  • class="p-"

    Prefix used for pages (ex. p-index, p-register) 

  • class="l-"

    Prefix used for page layout components (ex. l-header, l-main, l-footer, l-inner, l-sec etc) 

  • class="c-"

    Prefix used for all atomic components (ex. c-products-list, c-mini-cart-box, c-article-teaser etc) 

  • class="g-"

    Prefix used for grid utilities like margins, paddings etc (ex. g-spacing-top-2, g-gutter-bottom-4) 

  • class="u-"

    Prefix used for single utilities like: font styles (font-sizes, font-weights, text-transforms etc), backgrounds, colors, underlines etc (ex. u-text-upper, u-font-xxl, u-bg-1) 

  • class="t-"

    Prefix used for set of utilities making part of typography, like headings having common styles (ex. t-heading-1, t-heading-2) 

  • class="f-"

    Prefix used for form elements (ex. f-form, f-field) 

  • class="o-"

    Prefix used for objects like iframe, img, object (ex. o-object-wrapper, o-img-wrapper) 

  • class="h-"

    Prefix used for helpers (ex. h-show, h-hide, h-grid-flex-fix) 

  • class="js-"

    Prefix used for elements having JavaScript interactions ($('.js-example').on('click', function () {...})) 

  • class="is-/has-"

    Prefix used for component's state changed by JavaScript (ex: is-opened, is-active)

Useful Info

Some classes can be generated using 'gridle_generate_custom_class' method. 


@include gridle_generate_custom_class( ('u','-','text','-','centered','@','%state') ) {
    content: '#{gridle_get_current_state_name()}';
    text-align: center;
}

  • Text sizes

    Complexe modifiers with /@ rwd states

    • class="u-text-xxl"

      Look ma! I have 'xxl' size. I have my own rwd. It means I have 50px on desktop, 40px on tablet and 30px on mobile

    • class="u-text-xl"

      Look ma! I have 'xl' size. I have my own rwd. It means I have 30px on desktop, 25px on tablet and 20px on mobile

    • class="u-text-l"

      Look ma! I have 'l' size. I have my own rwd. It means I have 25px on desktop, 20px on tablet and 18px on mobile

    • class="u-text-xl u-text-l@mobile"

      Look ma! I have 'xl' size but i change my class to 'l' size on mobile by using '@mobile' state. It means on desktop and on tablet I have 'xl' size styles and on 'mobile' I have 'l' size styles. In this case I have 30px on desktop ('xl' styles for desktop), 25px on tablet ('xl' styles for tablet) and 18px on mobile ('l' styles for mobile)

    • class="u-text-xl u-text-l@tablet u-text-xl@mobile"

      Look ma! I have 'xl' size but i change my state to 'l' size on tablet by using '@tablet' state, and once again to 'xl' on mobile by using '@mobile' state. It means on desktop and on mobile I have 'xl' size styles and on tablet I have 'l' size styles. In this case I have 30px on desktop ('xl' styles for desktop), 20px on tablet ('l' styles for tablet) and 20px on mobile ('xl' styles for mobile). If I don't change state to 'xl' on mobile I would follow 'l' styles for mobile and have 18px
