Styleguide

Prefixes legend:

  • class="p-"

    Prefix used for pages (ex. p-index, p-register) 

  • class="l-"

    Prefix used for page layout components (ex. l-header, l-main, l-footer, l-inner, l-sec etc) 

  • class="c-"

    Prefix used for all atomic components (ex. c-products-list, c-mini-cart-box, c-article-teaser etc) 

  • class="g-"

    Prefix used for grid utilities like margins, paddings etc (ex. g-spacing-top-2, g-gutter-bottom-4) 

  • class="u-"

    Prefix used for single utilities like: font styles (font-sizes, font-weights, text-transforms etc), backgrounds, colors, underlines etc (ex. u-text-upper, u-font-xxl, u-bg-1) 

  • class="t-"

    Prefix used for set of utilities making part of typography, like headings having common styles (ex. t-heading-1, t-heading-2) 

  • class="f-"

    Prefix used for form elements (ex. f-form, f-field) 

  • class="o-"

    Prefix used for objects like iframe, img, object (ex. o-object-wrapper, o-img-wrapper) 

  • class="h-"

    Prefix used for helpers (ex. h-show, h-hide, h-grid-flex-fix) 

  • class="js-"

    Prefix used for elements having JavaScript interactions ($('.js-example').on('click', function () {...})) 

  • class="is-/has-"

    Prefix used for component's state changed by JavaScript (ex: is-opened, is-active)

Useful Info

LibSasserPlate by Pagepro use gridle.org system which allows to generate media queries states using 'gridle_register_state' method: 

  • desktop-small:
    @include gridle_register_state ( desktop-small , (
    max-width : $size_inner + $size_gutter*2
));
  • tablet:
    @include gridle_register_state ( tablet , (
    max-width : 1024px
));
  • mobile:
    @include gridle_register_state ( mobile , (
    max-width : 767px
));

RWD with gridle register states is pretty simple: 

  • RWD for exemplary 'some-class':
    .some-class {
    color: red;

    @include gridle_state (desktop-small) {
        color: green;
    }
    @include gridle_state (mobile) {
        color: blue;
    }
}
  • RWD for 'helper--text-xxl' mixin:
    @mixin helper--text-xxl () {
    font-size: 50px;
    line-height: 1;

    @include gridle_state (tablet) {
        font-size: 40px;
    }
    @include gridle_state (mobile) {
        font-size: 30px;
    }
}

Some classes can be generated using 'gridle_generate_custom_class' method: 

  • class="u-text-centered"
    @include gridle_generate_custom_class( ('u','-','text','-','centered','@','%state') ) {
    content: '#{gridle_get_current_state_name()}';
    text-align: center;
}
  • class="u-text-left"
    @include gridle_generate_custom_class( ('u','-','text','-','left','@','%state') ) {
    content: '#{gridle_get_current_state_name()}';
    text-align: left;
}

Simple gridle generated classes can be replaced one class with another with the same css properties. To replace class 'a' with class 'b', class 'b' must be followed by @ and by proper gridle register state from which the new class should replace the previous one, ex: 

  • class="u-text-centered u-text-right@tablet"

    Look ma! I'm centered on desktop and aligned to right on tablet and on mobile!

  • class="u-text-centered u-text-right@tablet u-text-left@mobile"

    Look ma! I'm centered on desktop, aligned to right on tablet and aligned to left on mobile!

Gridle generated classes can have also their own RWD styles, ex: 

  • RWD for griddle generated 'a' class:
    @include gridle_generate_custom_class( ('a','@','%state') ) {
    content: '#{gridle_get_current_state_name()}';

    color: red;

    @include gridle_state (desktop-small) {
        color: green;
    }
    @include gridle_state (tablet) {
        color: orange;
    }
    @include gridle_state (mobile) {
        color: blue;
    }
}
  • RWD for griddle generated 'b' class:
    @include gridle_generate_custom_class( ('b','@','%state') ) {
    content: '#{gridle_get_current_state_name()}';

    color: pink;

    @include gridle_state (desktop-small) {
        color: purple;
    }
    @include gridle_state (tablet) {
        color: yellow;
    }
    @include gridle_state (mobile) {
        color: aquamarine;
    }
}

Complexe gridle generated classes with their own RWD styles can be also replaced one class with another with the same css properties. To replace class 'a' with its own RWD with class 'b' with its own RWD, class 'b' must be followed by @ and by proper gridle register state for which the new class should replace the previous one, but in opposite to simple classes without their own RWD the new styles will replace the old ones only for listed gridle register states. It means if we have 4 griddle register states (desktop, desktop-small, tablet and mobile) and we want our element to have desktop and desktop-small styles from 'a' class but 'b' styles applied for tablet and mobile states we need to list all new @ states (class="a b@tablet b@mobile"). If we list only the proper gridle register state from which the new class should replace the previous one (in this case '@tablet') - our component will have 'a' styles for desktop, desktop-small and mobile and 'b' styles only for tablet, ex: 

  • class="a"

    Look ma! I have 'a' class styles with its own RWD. It means I'm red on desktop, green on desktop-small, orange on tablet and blue on mobile.

  • class="b"

    Look ma! I have 'b' class styles with its own RWD. It means I'm pink on dekstop, purple on desktop-small, yellow on tablet and aquamarine on mobile.

  • class="a b@tablet"

    Look ma! On desktop, desktop-small and mobile I have 'a' class styles with its own RWD but on tablet I change my styles for 'b' class with its own RWD. It means I'm red on desktop, green on desktop-small and blue on mobile (like 'a' class elements) but on tablet I'm yellow (like 'b' class elements)

  • class="a b@tablet b@mobile"

    Look ma! On desktop and desktop-small I have 'a' class styles with its own RWD but on tablet and on mobile I change my styles for 'b' class with its own RWD. It means I'm red on desktop and green on desktop-small (like 'a' class elements) and I'm yellow on tablet and aquamarine on mobile (like 'b' class elements)

'h-' Helper classes examples:

  • Grid fix

    Regular class without /@ gridle register states and without their own RWD styles. This class can be applied to gridle.org 'grid' element to fix css style bug (griddle.org 'grid' element has flex styles applied but also 'display: block' which eliminates flex properties. Also it's often very useful to change 'grid' element flex-direction styles from column to row).

    • class="h-grid-flex-fix"
      .h-grid-flex-fix {
    display: flex;
    flex-direction: row;
}"
